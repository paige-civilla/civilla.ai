GOAL
Add a new authenticated module: Disclosures & Discovery (and a dashboard tile).
This module must:
1) Explain “Disclosures vs Discovery” in UPL-safe, educational language.
2) Show the state-specific naming label for the case state (Idaho/Florida/California/Arizona mapped; all others fallback).
3) Provide full CRUD for:
   A) Disclosure checklist items (what must be exchanged + status + due date)
   B) Discovery log items (requests served/received + type + due date + status)
4) Keep everything per-user and per-case (userId + caseId enforced like other modules).

ROUTES / URLS
- Canonical route: /app/disclosures/:caseId
- Also accept /app/discovery/:caseId as an alias redirecting to /app/disclosures/:caseId (so URL matches user expectation).
- Add navbar + dashboard tile entry named “Disclosures & Discovery”.

DATABASE (shared/schema.ts)
Create two tables:

1) case_disclosure_items
- id (serial pk)
- userId (text, not null)
- caseId (int, not null)
- title (text, not null)
- category (text, not null)  // e.g., "Financial", "Parenting", "Other"
- status (text, not null default "pending") // pending | in_progress | complete
- dueDate (date nullable)
- notes (text nullable)
- createdAt, updatedAt (timestamps)

2) case_discovery_items
- id (serial pk)
- userId (text, not null)
- caseId (int, not null)
- direction (text not null default "sent") // sent | received
- requestType (text not null) // interrogatories | rfp | rfa | subpoena | other
- title (text not null)
- servedDate (date nullable)
- dueDate (date nullable)
- status (text not null default "open") // open | responded | closed
- relatedContactId (int nullable) // optional link to contacts
- notes (text nullable)
- createdAt, updatedAt (timestamps)

Add zod insert/update schemas similar to tasks/deadlines patterns.

DB INIT (server/db.ts)
Add runtime table creation statements for both tables (same style as other modules).

MODULE INTRO COPY (paste verbatim)
Title: Disclosures & Discovery
Body:
“Most family courts require some form of initial (sometimes ‘mandatory’) disclosures—basic documents and information exchanged early in the case. Discovery is different: it’s the process of requesting and responding to specific information (interrogatories, requests for production, subpoenas, and more). Use this module to track what’s required, what was sent or received, and what’s complete. Terminology and exact requirements vary by state; Civilla labels this section using your case’s state when possible.”
