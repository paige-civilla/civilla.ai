TABLE: trial_prep_shortlist
Purpose:
A lightweight “trial binder” shortlist that can pull the best 1–3 items per section later, without needing the full Trial Prep module yet.

Columns:
- id (serial/uuid consistent with schema.ts patterns)
- userId (text, not null, indexed)
- caseId (int, not null, indexed)
- sourceType (text enum): "evidence" | "evidence_note" | "timeline_event" | "communication" | "document" | "exhibit_item"
- sourceId (int, not null)  // points to the source record id
- title (text, not null)    // user-facing label shown in the shortlist
- summary (text, nullable)  // short context snippet
- binderSection (text, not null, default "General")
  Allowed values (use as a string union in TS, but store as text):
  "Overview"
  "Key Facts"
  "Key Evidence"
  "Timeline Highlights"
  "Communications"
  "Discovery & Disclosures"
  "Witnesses"
  "Financial"
  "Parenting"
  "Safety"
  "Other"
- importance (int, not null, default 3) // 1=highest, 5=lowest
- tags (json array of strings, not null, default '[]')
- color (text, nullable) // optional color chip for the card
- isPinned (boolean, not null, default false)
- createdAt (timestamp, not null, default now)
- updatedAt (timestamp, not null, default now)

Constraints:
- Index on (userId, caseId)
- Index on (caseId, binderSection)
- Optional uniqueness to prevent duplicates:
  UNIQUE(userId, caseId, sourceType, sourceId)

- shared/schema.ts: add drizzle table + zod insert/update schemas

ROUTES (requireAuth, verify case ownership):
