REPLIT TASK: Phase 2E.1 — Auto-name compiled documents + inline rename UI (Document Creator polish)

Goal
When a user compiles a document from Claims, Civilla should:
1) Auto-name the document consistently
2) Allow quick inline rename after creation (like Lexi thread rename)
3) Keep names readable and ordered

A) Backend — Auto-name on compile (server/routes.ts)
Find the compile endpoint:
POST /api/cases/:caseId/documents/compile-claims

Update it so it creates the Document with a default title when none is provided:
Default title format:
"<Draft Topic> — <Case Title> — <YYYY-MM-DD>"

Rules:
- draftTopic comes from request body (already added in Phase 2E)
- if draftTopic missing: use "Compiled Summary"
- case title from cases table
- date should be server-local date in ISO YYYY-MM-DD

Also return the created document id + title:
Response should include:
{ ok: true, document: { id, title, ... } }
